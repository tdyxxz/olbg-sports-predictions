# Backtest Starter

This folder is the normalized backtest setup for the five target sports:

- `Snooker`
- `Greyhounds`
- `Motor Racing`
- `Rugby Union`
- `Golf`

## Files

- `historical_markets_template.csv`
  Normalized event/market/outcome data.
- `strategy_picks_template.csv`
  The selections our strategy made.
- `backtest.py`
  Joins picks to market results and calculates profitability.

## What We Still Need

For a real profitability test, each pick needs:

- an `event_id`
- a `market`
- a `selection`
- the odds available at decision time, ideally closing odds
- the settled result (`win`, `loss`, `push`, `void`)

Free sources that look useful for historical results:

- Snooker: [snooker.org](https://www.snooker.org/)
- Formula 1 results: [Ergast F1](https://openpublicapis.com/api/ergast-f1)
- Greyhound results/form: [Racing Post GreyhoundBet archive article](https://www.racingpost.com/greyhounds/full-replay-archive-live-on-greyhoundbet-aLSoS6v7NPjj/) and [Racing and Sports greyhound results](https://www.racingandsports.co.uk/greyhound-racing-results/united-kingdom)
- Golf result archives: [Livesport golf archives](https://www.livesport.com/en/golf/)
- Rugby union results/stats: [RugbyPass](https://www.rugbypass.com/)

Important limitation:

- I have not found a single free, comprehensive, reliable source for historical closing odds across all five sports.
- That means free outcome backtesting is feasible; true odds-based ROI backtesting is only partially feasible until we have usable historical prices.

## CSV Workflow

1. Fill `historical_markets_template.csv` with one row per selectable market outcome.
2. Fill `strategy_picks_template.csv` with one row per pick generated by the strategy.
3. Run:

```powershell
python backtest\backtest.py `
  --markets backtest\historical_markets_template.csv `
  --picks backtest\strategy_picks_template.csv
```

## Normalization Rules

- Use decimal odds in `odds_decimal`.
- Use one row per outcome, not one row per event.
- Keep `event_id` stable across all rows belonging to the same event.
- Use these `result` values only:
  - `win`
  - `loss`
  - `push`
  - `void`
- Use sport names exactly:
  - `Snooker`
  - `Greyhounds`
  - `Motor Racing`
  - `Rugby Union`
  - `Golf`

## Snooker EV Workflow

The generic starter above is useful for settled picks, but it is not enough to optimize a snooker betting strategy. For snooker, the important question is not "who won?" but "did our model beat the market?"

Use these added files for that:

- `snooker_schema.sql`
  SQLite schema for a pre-match snooker dataset.
- `snooker_value_template.csv`
  Flat-file template with the minimum columns needed for an EV backtest.
- `snooker_backtest.py`
  Reads one row per match, converts closing odds to no-vig probabilities, chooses the higher-edge side, and measures ROI plus CLV-style price improvement.
- `snooker_collect.py`
  Pulls event and match history from the snooker.org API and normalizes it.
- `snooker_results_template.csv`
  Manual results template if you want to start without any API access.
- `snooker_build_dataset.py`
  Builds pre-match features, merges odds, and writes a backtest-ready CSV.
- `snooker_train.py`
  Fits a lightweight logistic model from the feature CSV and writes updated probabilities.
- `snooker_threshold_sweep.py`
  Tests many edge/probability settings so you can see which thresholds were historically profitable.
- `snooker_prediction_prompt_v4.txt`
  A prompt for upcoming matches that matches the backtested value-betting logic.
- `snooker_odds_template.csv`
  Template for historical prices.
- `snooker_rankings_template.csv`
  Optional ranking input file.

### Snooker CSV Rules

- `model_prob_a` must be your model's pre-match win probability for `player_a`.
- `close_odds_a` and `close_odds_b` should be the closing decimal odds.
- `price_taken_a` and `price_taken_b` are optional but recommended.
  If blank, the script assumes you bet at close.
- `winner` must be either `player_a` or `player_b`.
- Keep all features pre-match only.
- The included feature builder writes a baseline `model_prob_a` so the pipeline runs end to end.
  Treat that number as a placeholder until you replace it with a trained or calibrated model.

### What The Script Does

For each match it:

1. converts the closing two-way market into no-vig probabilities
2. compares your model probability to the market probability for both sides
3. selects the side with the larger positive edge
4. filters out bets that do not meet your edge and model-probability thresholds
5. settles profit using the price you actually took

### Example Command

```powershell
python backtest\snooker_backtest.py `
  --data backtest\snooker_value_template.csv `
  --min-edge 0.04 `
  --min-model-prob 0.54 `
  --stake 1.0 `
  --export backtest\snooker_bets_out.csv
```

### Beginner Path Without Contacting Any Support Service

This is the easiest way to start:

1. Use [CueTracker](https://cuetracker.net/) as your results source.
2. Use [OddsPortal](https://www.oddsportal.com/snooker/results/) or [BetExplorer](https://www.betexplorer.com/snooker/) as your odds source.
3. Manually enter or paste the historical rows into the CSV templates in this folder.
4. Run the builder, trainer, and backtester locally.

This avoids the snooker.org header requirement entirely.

4. Build the feature dataset:

```powershell
python backtest\snooker_build_dataset.py `
  --results backtest\snooker_results_template.csv `
  --odds backtest\snooker_odds_template.csv `
  --rankings backtest\snooker_rankings_template.csv `
  --output backtest\snooker_value_built.csv
```

5. Train the probability model:

```powershell
python backtest\snooker_train.py `
  --data backtest\snooker_value_built.csv `
  --train-ratio 0.8 `
  --weights-out backtest\snooker_model_weights.json `
  --predictions-out backtest\snooker_value_scored.csv
```

6. Backtest the value rules:

```powershell
python backtest\snooker_backtest.py `
  --data backtest\snooker_value_scored.csv `
  --min-edge 0.04 `
  --min-model-prob 0.54
```

7. Sweep thresholds to find the best historical settings:

```powershell
python backtest\snooker_threshold_sweep.py `
  --data backtest\snooker_value_scored.csv `
  --edge-start 0.02 `
  --edge-stop 0.08 `
  --edge-step 0.01 `
  --prob-start 0.50 `
  --prob-stop 0.62 `
  --prob-step 0.02 `
  --min-bets 10 `
  --export backtest\snooker_threshold_results.csv
```

### Train The Probability Model

Once you have a larger historical CSV, replace the placeholder probabilities with learned ones:

```powershell
python backtest\snooker_train.py `
  --data backtest\snooker_value_built.csv `
  --train-ratio 0.8 `
  --weights-out backtest\snooker_model_weights.json `
  --predictions-out backtest\snooker_value_scored.csv
```

Then backtest the scored file:

```powershell
python backtest\snooker_backtest.py `
  --data backtest\snooker_value_scored.csv `
  --min-edge 0.04 `
  --min-model-prob 0.54
```

### Results Source Notes

- You can skip `snooker_collect.py` entirely if you do not want to use the snooker.org API.
- The simpler beginner method is to fill `snooker_results_template.csv` from CueTracker manually.
- This is slower but much easier to understand and debug when you are starting.

### Odds Source Notes

- Use OddsPortal first for historical prices.
- Use BetExplorer as backup when a match is missing.
- Keep bookmaker naming and source links in the odds CSV if you want an audit trail, but they are not required for the model to run.

### Feature Builder Notes

`snooker_build_dataset.py` derives:

- last-5 wins
- last-5 frame differential
- in-tournament wins and frame differential
- rest-day differences
- previous-match decider flags
- head-to-head wins over the previous 3 years

It also merges odds and rankings, then produces a baseline probability estimate from those features. That baseline is intentionally simple and should be replaced later with a trained model or calibrated probability mapping.

### Training Notes

`snooker_train.py` uses:

- a time-based split, not a random split
- standardized feature values
- logistic regression trained with gradient descent
- plain standard-library Python only

It prints train and test metrics for:

- log loss
- Brier score
- accuracy

It also writes:

- a JSON file with learned weights and scaling parameters
- an updated CSV where `model_prob_a` comes from the learned model

This is intentionally lightweight. It is not meant to be the final model, but it gives you a real probability-learning step so the backtest is optimizing edge instead of hand-tuned confidence scores.

### Finding Profitability

You cannot know whether the strategy is profitable until you run the sweep on real historical matches with real odds.

The file to look at after that is:

- `snooker_threshold_results.csv`

The most useful columns are:

- `roi_pct`
- `profit`
- `bets`
- `avg_edge_pct`
- `avg_clv_pct`

Do not trust a result built on only a handful of bets.
As a rule of thumb, prefer settings that are profitable across a decent sample size rather than the single highest ROI row with very few bets.

### What To Do First If You Are A Beginner

Do not try to build a full season immediately.

Start with just 20 to 30 past matches:

1. fill `snooker_results_template.csv`
2. fill `snooker_odds_template.csv`
3. run `snooker_build_dataset.py`
4. run `snooker_train.py`
5. run `snooker_backtest.py`

Once that works, expand to 100+ matches.

### Suggested First Thresholds

- `min-edge = 0.04`
- `min-model-prob = 0.54`
- `max-short-price = 1.45`
- `short-price-edge = 0.08`

These are deliberately conservative. Once you have a larger sample, sweep the thresholds and compare:

- ROI
- average edge
- CLV percentage
- win rate
- number of bets

### Best Next Build Step

Once the CSV pipeline is working, the next upgrade should be a feature builder that automatically derives:

- last-5 and last-10 win rates
- last-5 and last-10 frame differential
- recent H2H

That is the point where you can stop relying on hand-entered model probabilities and start training or calibrating a real probability model.
